## 四、Tokenization 详解

### 4.1 什么是 Tokenizer

Token 是文本和整数之间的桥梁：

```
文本 → Tokenizer.encode() → [整数序列] → Tokenizer.decode() → 文本
```

核心直觉：**把字符串拆成高频出现的片段（subword）**。

### 4.2 为什么需要 Tokenization

- 直接使用原始字节（byte-level）在概念上更优雅，但以当前架构来看是 compute-inefficient
- Tokenizer-free 方法（ByT5, MegaByte, BLT, T-Free）有前景，但还没 scale 到前沿水平
- 所以当前主流做法：BPE（Byte-Pair Encoding）

### 4.3 BPE（Byte-Pair Encoding）

BPE 的核心思想：
1. 从字符级别开始（每个字符是一个 token）
2. 统计所有相邻 token pair 的出现频率
3. 合并最高频的 pair，生成一个新的 token
4. 重复直到达到目标 vocab size

**BPE 训练过程：**
- 输入：大量文本语料 + 目标 vocab size
- 输出：一个 merge list（merge 规则的顺序列表）
- 这个 merge list 就是 tokenizer 的"词典"

**BPE 编码过程：**
- 对输入文本，按 merge list 的顺序依次应用所有 merge 规则
- 最终得到 token 序列

**BPE 解码过程：**
- 直接查表，把 token id 映射回字符串片段并拼接

### 4.4 Tokenizer 的抽象接口

```python
class Tokenizer(ABC):
    def encode(self, string: str) -> list[int]:
        raise NotImplementedError

    def decode(self, indices: list[int]) -> str:
        raise NotImplementedError
```

### 4.5 对 Coding Agent 的直接影响

- 文件路径、函数名、错误日志、JSON、diff/patch 都经过 tokenizer
- Token 切分方式会影响：
  - 复制标识符的稳定性
  - 生成 diff 的精确度
  - 解析错误日志的能力
  - 长上下文的 token 效率
