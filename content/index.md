# Coding Agent Model and Memory Research OS

我的目标是用一年左右的时间，沿着一条清晰主线持续积累：

> 以模型后训练为主线，以 Coding Agent Memory 为研究问题，以 Coding Agent 为工程落地场景。

最终希望形成的能力不是“学完所有 AI”，而是能把真实 Coding Agent 的失败转化成可训练的数据、可验证的 reward、可复现的 eval 和有说服力的实验报告。

## 总览

总览专题说明这条路线为什么这样设计，以及几条能力线之间的关系。

它回答三个问题：我要学什么、为什么这些内容放在一起、每个阶段应该产出什么。Agent Training、Memory 和科研提升都收在这个专题下面，作为整体路线的三个组成部分。

## 周计划

周计划负责把长期目标拆成可执行的一周。

每周有一个独立目录：本周计划写清楚 A 线、B 线、每日安排和交付物；每天学习完成后，再更新对应日期页，记录当天进度、卡点、产出和第二天调整。

## 专题

专题用于沉淀深入调研内容。

例如 Multica Memory 调研、episodic / semantic memory、某篇重要论文或某个框架的代码阅读，都不应该塞进周计划正文，而应该成为独立专题。周计划只负责安排什么时候读、读到什么程度、产出什么判断。

## 项目

项目页用于记录中长期工程方向。

小实验代码放在 `labs/`；成熟工程单独建 GitHub repo。这个站点只维护项目动机、实验报告、关键结论和外部链接，避免把知识系统变成混乱的代码仓库。

## 模板

模板用于保证长期维护时格式稳定。

目前包括周计划、每日进度、论文笔记、实验记录和周复盘模板。后续新增内容时优先用模板生成，而不是手动从零写。
