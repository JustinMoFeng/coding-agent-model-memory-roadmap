# 科研能力提升

## 目标

这条线服务读论文、写论文、做工程和做实验。

核心目标是：

> 把学习和工程现象逐步转化为可复现的实验、清晰的研究问题和可展示的技术报告。

## 每周系统

每周至少留下：

- 一份 A 线学习笔记。
- 一份 B 线调研笔记。
- 一个小实验、代码片段或结构图。
- 一份周复盘。

每日进度不要写在周计划正文里，而是写到当周目录下的日期页。

## 论文卡片

每篇论文不要只写 summary，要回答：

- Problem 是什么？
- Method 是什么？
- Data 怎么来？
- Objective / reward 是什么？
- Eval 是否可信？
- 对 Coding Agent post-training 或 Memory 有什么启发？
- 最小复现实验是什么？

模板见 [论文笔记模板](../reference/templates/paper-note.md)。

## 实验记录

每个实验记录：

- Question。
- Hypothesis。
- Setup。
- Procedure。
- Results。
- Failure cases。
- Decision。
- Next step。

模板见 [实验记录模板](../reference/templates/experiment-log.md)。

## 代码组织

小实验代码放 `labs/`。

成熟项目单独建 repo，并在这里维护：

- 项目说明。
- 实验报告。
- 关键结果。
- 外部链接。

详细原则见 [知识系统与代码组织](../archive/08-research-os-and-code-organization.md)。
