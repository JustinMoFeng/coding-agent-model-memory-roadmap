# Week XX：标题

## 1. 介绍

### 简单介绍

用 1-2 段说明本周主题、为什么安排这一周，以及它和整体路线的关系。

### A 线要学什么

说明模型 / 后训练线本周要学什么。

建议包含：

- 本周核心问题。
- 主要学习材料。
- 本周做到什么程度。

### B 线要学什么

说明 Memory / 工程调研线本周要学什么。

建议包含：

- 本周核心问题。
- 主要阅读材料或代码路径。
- 本周做到什么程度。

### 本周不做什么

列出本周明确不做的内容，避免范围失控。

原则：介绍区只写方向和边界，具体每日任务放到第 2 部分。

## 2. 每日安排

按天写，每天包含三块：

```text
### Day N：主题

A 线：
- ...

B 线：
- ...

产出：
- ...
```

原则：

- 每天任务要能在 1-1.5 小时内推进。
- A 线和 B 线都要具体到材料、代码、实验或问题。
- 产出必须可检查，不能只写“学习”“理解”。
- 详细背景不要塞进周计划，放到专题文档并链接。

## 3. 每周交付

列出本周必须提交或沉淀的 artifacts。

建议至少包含：

- 一份 A 线学习笔记。
- 一份 B 线调研笔记或代码阅读笔记。
- 一个小实验、代码片段或结构图。
- 一份周复盘。

## 4. 本周在整体路线中的作用

用 2-4 段说明：

- 本周为什么是当前阶段需要的。
- 它如何连接上一周。
- 它如何铺垫下一周。
- 它如何服务长期目标：模型后训练、Coding Agent Memory、工程能力和学术写作。
